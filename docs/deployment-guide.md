# Deployment Guide

## Prerequisites

- AWS CLI configured with appropriate permissions in the management/central account
- CloudFormation StackSets enabled for your AWS Organization
- CloudTrail enabled in all member accounts (management events)

## Step 1: Deploy the Central Account Stack

```bash
aws cloudformation deploy \
  --template-file central-account-template.yaml \
  --stack-name service-quota-monitoring-central \
  --parameter-overrides \
    OrganizationId=o-YOUR-ORG-ID \
    AlertEmailAddress=your-team-dl@yourcompany.com \
    EnableAccountNameResolution=true \
    EnableSupportCaseLookup=false \
  --capabilities CAPABILITY_NAMED_IAM \
  --region us-east-1
```

Replace `o-YOUR-ORG-ID` with your actual AWS Organization ID and update the email address.

## Step 2: Confirm the SNS Email Subscription

Check the alert email inbox for a subscription confirmation email from AWS SNS. **You must click the confirmation link** before alerts will be delivered.

## Step 3: Retrieve the Central Event Bus ARN

```bash
CENTRAL_BUS_ARN=$(aws cloudformation describe-stacks \
  --stack-name service-quota-monitoring-central \
  --query 'Stacks[0].Outputs[?OutputKey==`CentralEventBusArn`].OutputValue' \
  --output text \
  --region us-east-1)

echo $CENTRAL_BUS_ARN
```

## Step 4: Deploy the Member Account Template via StackSet

```bash
aws cloudformation create-stack-set \
  --stack-set-name service-quota-monitoring-member \
  --template-body file://member-account-template.yaml \
  --parameters ParameterKey=CentralEventBusArn,ParameterValue=$CENTRAL_BUS_ARN \
  --capabilities CAPABILITY_NAMED_IAM \
  --permission-model SERVICE_MANAGED \
  --auto-deployment Enabled=true,RetainStacksOnAccountRemoval=false

aws cloudformation create-stack-instances \
  --stack-set-name service-quota-monitoring-member \
  --deployment-targets OrganizationalUnitIds=ou-YOUR-OU-ID \
  --regions us-east-1 us-west-2 eu-west-1 \
  --operation-preferences MaxConcurrentPercentage=25,FailureTolerancePercentage=10
```

Replace `ou-YOUR-OU-ID` with your target Organizational Unit ID(s). Adjust the regions list to match where your workloads run.

## Step 5: Validate

After deployment, trigger a test Service Quota Increase request in any member account and verify you receive the formatted email alert within minutes.

## Troubleshooting

| Symptom | Solution |
|---------|----------|
| No email received | Confirm SNS subscription (Step 2) |
| Events not forwarded | Verify CloudTrail is enabled in the member account |
| Lambda error | Check CloudWatch Logs for `/aws/lambda/ServiceQuotaIncreaseAlert` |
| Account name shows as "Account 123..." | Ensure Lambda role has `organizations:DescribeAccount` permission and `EnableAccountNameResolution=true` |
